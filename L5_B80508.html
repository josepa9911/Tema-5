L5 - Cadenas de Markov¶

El caso especial de los procesos aleatorios donde existen "estados" y "transiciones" entre estos estados, las cadenas de Markov modelan las probabilidades de transicin y el tiempo de permanencia entre cada estado. En particular, la teora de colas describe la evolucin del nmero de elementos en la fila, dado un flujo de entrada y un flujo de salida.

Sistemas M/M/1¶

Con una tasa de llegada $\lambda$ y un parmetro de tiempo de servicio $\nu$, un proceso de Markov con un solo servidor tambin recibe la notacin de sistema de colas M/M/1 (donde la M viene de Markov). Ms en general, con $s$ servidores es un sistema M/M/s.

La simulacin de un sistema del tipo M/M/1 implica la generacin de una llegada de "clientes" como una corriente de Poisson. Esto es equivalente a decir que tienen una distribucin de probabilidad de tiempo entre arribos con distribucin exponencial y parmetro $\lambda$.

Por su parte, el tiempo de servicio tiene tambin una distribucin exponencial pero con parmetro $\nu$. A la relacin $\lambda/\nu$ usualmente se le conoce como $\rho$.

Nota: El tiempo de servicio se asume independiente del tiempo de llegada.

Sobre la simulación¶

Es posible crear una simulacin de $N$ clientes con sus respectivos tiempos de servicio, distribuidos en el tiempo.

Para medir el tiempo se puede utilizar una medida mnima arbitraria, pero que tenga sentido para el problema. Por ejemplo: si la tasa de llegada es de 1 persona/minuto, vale ms tener una granularidad de segundos o decenas de segundos para capturar una precisin temporal adecuada.

Nota: De la seleccin de $N$ puede depender la precisin deseada para el resultado. Por ejemplo: con $N = 1000$ es posible obtener una precisin de hasta el 0,001 o 0,1% ($1/N$).

Problema de ejemplo: un servidor web¶

Un servidor web es modelado como un sistema M/M/1 con una tasa de arribo de 2 solicitudes por minuto. Es deseado tener 4 o menos solicitudes en fila el 99\% del tiempo. Qu tan rpido debe ser el servicio? $\nu$ es solicitudes atendidas por minuto.

El estado $i$ es el nmero de clientes en el sistema. La longitud de la fila es $L_q = i - 1$ (queue length), en virtud de la solicitud que est siendo atendida en $s = 1$ servidores. Es posible encontrar que:

$$ P( \text{5 o ms clientes en el sistema} ) = \sum_{i=5}^{\infty} (1 - \rho) \rho^i = 1 - \sum_{i=0}^{4} (1 - \rho) \rho^i = \rho^5 $$

que depende de $\rho = \lambda/\nu$ y del parmetro de servicio $\nu$ buscado.

De los datos del problema: $\lambda = 2$. Para tener una fila de 3 o menos clientes el 99\% del tiempo se necesita:

$$ \begin{aligned} P( \text{5 o ms clientes en el sistema} ) = \rho^5 & = \left( \frac{\lambda}{\nu} \right)^5 \leq 0.01 \\ \nu^5 & \geq \frac{\lambda^5}{0.01} = \frac{2^5}{0.01} = 3200 \quad \Rightarrow \quad \nu \geq 5.024 \end{aligned} $$

es decir, el servidor debe atender ms de 5,024 solicitudes por minuto en promedio para poder satisfacer el requisito.

En la siguiente simulacin, con $N = 1000$, y $\nu = 3 < 5.024$ deberamos obtener una probabilidad $P( \text{5 o ms clientes en el sistema} ) > 0.01$ que no cumple con las especificaciones.

Nota: Observar el cambio de unidades de minutos a segundos, para lograr mayor "granularidad".

import numpy as np
from scipy import stats
import matplotlib.pyplot as plt

# Nmero de clientes
N = 1000

# Parmetro de llegada (clientes/segundos)
lam = 2/60

# Parmetro de servicio (servicios/segundos)
nu = 3/60

# Distribucin de los tiempos de llegada entre cada cliente
X = stats.expon(scale = 1/lam)

# Distribucin de los tiempos de servicio a cada cliente
Y = stats.expon(scale = 1/nu)

# Intervalos entre llegadas (segundos desde ltimo cliente)
t_intervalos = np.ceil(X.rvs(N)).astype('int')

# Tiempos de las llegadas (segundos desde el inicio)
t_llegadas = [t_intervalos[0]]
for i in range(1, len(t_intervalos)):
    siguiente = t_llegadas[i-1] + t_intervalos[i]
    t_llegadas.append(siguiente)

# Tiempos de servicio (segundos desde inicio de servicio)
t_servicio = np.ceil(Y.rvs(N)).astype('int')

# Inicializacin del tiempo de inicio y fin de atencin
inicio = t_llegadas[0]          # primera llegada
fin = inicio + t_servicio[0]    # primera salida

# Tiempos en que recibe atencin cada i-simo cliente (!= que llega)
t_atencion = [inicio]
for i in range(1, N):
    inicio = np.max((t_llegadas[i], fin))
    fin = inicio + t_servicio[i]
    t_atencion.append(inicio)

# Inicializacin del vector temporal para registrar eventos
t = np.zeros(t_atencion[-1] + t_servicio[-1] + 1)

# Asignacin de eventos de llegada (+1) y salida (-1) de clientes
for c in range(N):
    i = t_llegadas[c]
    t[i] += 1
    j = t_atencion[c] + t_servicio[c]
    t[j] -= 1

# Umbral de P o ms personas en sistema (hay P - 1 en fila)
P = 5

# Instantes (segundos) de tiempo con P o ms solicitudes en sistema
frecuencia = 0

# Proceso aleatorio (estados n = {0, 1, 2...})
Xt = np.zeros(t.shape)

# Inicializacin de estado n
n = 0

# Recorrido del vector temporal y conteo de clientes (estado n)
for i, c in enumerate(t):
    n += c # sumar (+1) o restar (-1) al estado
    Xt[i] = n
    if Xt[i] >= P: 
        frecuencia += 1

# Fraccin de tiempo con P o ms solicitudes en sistema
fraccion = frecuencia / len(t)

# Resultados
print('Parmetro lambda =', str(lam*60))
print('Parmetro nu =', str(nu*60))
print('Tiempo con ms de {} solicitudes en fila:'.format(P-2))
print('\t {:0.2f}%'.format(100*fraccion))
if fraccion <= 0.01:
    print('\t S cumple con la especificacin.')
else:
    print('\t No cumple con la especificacin.') 
print('Simulacin es equivalente a {:0.2f} horas.'.format(len(t)/3600))

# Grfica de X(t) (estados del sistema)
plt.figure()
plt.plot(Xt)
plt.plot(range(len(t)), (P-1)*np.ones(t.shape))
plt.legend(('$X(t) = n$', '$L_q = $' + str(P-2)))
plt.ylabel('Clientes en el sistema, $n$')
plt.xlabel('Tiempo, $t$ / segundos')
plt.show()

Simulacin es equivalente a 8.62 horas.

Asignaciones¶

La administracin del servicio desea ahora que el servidor no est vaco (sin atender solicitudes) ms del 10% del tiempo. Hallar el parmetro $\nu$ para satisfacer este requisito y modificar el cdigo para medir la variable de inters en una simulacin.

Análisis de la asignación¶

La asignacin consiste en que con base en el sistema M/M/1 descrito anteriormente, se disee un nuevo sistema que cumpla la condicin de que el servidor web no est vaco ms del 10% del tiempo. Otro forma de ver tal condicin, es que el 90% del tiempo haya 1 o ms clientes en el servidor. Para este caso nuevamente se utilizar el parmetro $\lambda$ para representar la llegada de clientes por minutos y de la misma manera tendr un valor de 2. Entonces, se procede a la parte de formar el servidor con las condiciones mencionadas anteriormente y para ello se ha de obtener el parmetro $\nu$ que las satisfaga. Por tanto, con base en las ecuaciones anteriores, se determinar el valor de $\nu$; tomando P como la probabilidad de que en en el servidor web est presente uno o ms clientes. Las frmulas seran:

El estado $i$ es el nmero de clientes en el sistema.

$$ P( \text{1 o ms clientes en el sistema} ) = \sum_{i=1}^{\infty} (1 - \rho) \rho^i = 1 - \sum_{i=0}^{1} (1 - \rho) \rho^i = \rho $$

que depende de $\rho = \lambda/\nu$ y del parmetro de servicio $\nu$ buscado.

De los datos del problema: $\lambda = 2$. $$ \begin{aligned} P( \text{1 o ms clientes en el sistema} ) = \rho & = \frac{\lambda}{\nu} \geq 0.9 \\ \nu & \leq \frac{\lambda}{0.90} = \frac{2}{0.90} = 2.22222 \quad \end{aligned} $$

Esta ecuacin anterior, en esencia dicta que la probabilidad de que haya 1 o ms clientes en el servidor ha de ser idealmente mayor al 90%; en caso de no ser mayor del 90% indicara que el servidor tuvo ms de 10% del tiempo total sin tener clientes para atender, lo que infringira las condiciones propuestas en el enunciado. Finalmente, se desear corroborar el resultado hallado en la inequacin sobre $\nu$ con distintos valores, puesto que simplemente que $\nu$ ha de ser menor a 2.2222 ; lo que indicara que para valores menores al anterior es mucho ms probable cumplir con las condiciones del enunciado mientras que para valores mayores al anterior se tiene mayor facilidad de fallar lo pedido en el enunciado.

Conclusión de la simulación¶

Se utilizaron distintos valores de nu para corroborar lo mencionado anteriromente. Es claro destecar que $\nu$ representa el servicio del servidor web hacia los clientes, entonces se nos indic mediante las ecuaciones que para una llegada de 2 clientes por minuto, se ha de necesitar al menos 2.222 clientes por segundo para poder mantener un 10% del tiempo total como mximo sin antender clientes. Cabe destacar que al realizar la simulacin con un valor de $\nu$ de 2.2222 no asegura un 10% exacto con 0 clientes atendidos, sino ya que al trabajar con distribuciones de variables aleatorias se puede predecir que con ese valor es el necesario para poder ajustar las condiciones requeridas. Entonces, al simular comparando los distintos valores $\nu$ se puede apreciar directamente que para los valores menores de 2.2222 s se cumpli que haya al menos un 10% del tiempo sin clientes para atender, y a medida que ms cercano el valor de $\nu$ sea a cero, menor ser el porcentaje de tiempo sin clientes. En cambio al incrementar el valor de $\nu$ quiere decir que son ms capaces de atender clientes por minutos, por tanto el tiempo sin clientes para atender se disparara ya que por ejemplo, con $\nu = 5 \quad y \quad \lambda = 2$, se interpreta que llegan 2 clientes por minuto, pero se es capaz de atender en el servidor web a 5 clientres por minuto; entonces claramente se incrementara por mucho el tiempo sin clientes en el servidor.
